# Ralph Loop Template

Claude Code による自律的タスク駆動開発のテンプレートリポジトリ。

## Ralph Loop とは

Ralph Loop は、Claude Code を「毎回セッションをリセットしながら 1 タスクずつ自律的に進める」開発手法。ファイルシステムを長期記憶として使い、コンテキスト汚染と方針固着を防ぐ。

## 使い方

### 1. テンプレートからリポジトリを作成

GitHub の「Use this template」ボタンで新リポジトリを作成するか、手動でコピー：

```bash
git clone https://github.com/yourname/ralph-loop-template.git my-project
cd my-project
rm -rf .git && git init
```

### 2. プロジェクトに合わせてファイルを編集

1. **`docs/PRD.md`** — プロジェクトの PRD を書く（テンプレートに沿って埋める）
2. **`CLAUDE.md`** — プロジェクト固有のルール・アーキテクチャを記述
3. **`PROMPT.md`** — 必要に応じてビルドコマンド等を調整
4. **`tasks.md`** — PRD の Phase 1 のタスクを展開

### 3. 実行

```bash
# 手動で 1 イテレーションずつ確認
./ralph.sh --once

# 自動ループ（全タスク完了 or 上限到達で停止）
./ralph.sh

# 最大 30 イテレーションで停止
./ralph.sh -n 30
```

## ファイル構成

```
.
├── README.md          # このファイル
├── ralph.sh           # ループ実行スクリプト
├── PROMPT.md          # 毎イテレーションで Claude Code に渡すプロンプト
├── tasks.md           # タスク管理（チェックリスト形式）
├── CLAUDE.md          # Claude Code 設定（プロジェクト概要・ルール）
├── docs/
│   └── PRD.md         # プロジェクト要件定義書テンプレート
└── .ralph-logs/       # 実行ログ（自動生成）
```

## 設計思想

### なぜセッションリセットが有効か

1. **ファイルシステムが長期記憶** — tasks.md + ソースコード + CLAUDE.md がプロジェクト状態を完全に表現。会話履歴に依存しない
2. **リセットがむしろ強み** — コンテキスト汚染と方針固着を防ぎ、毎回ソースコードの実態から判断し直せる
3. **タスク粒度の設計** — 1 タスクが 1 セッションのコンテキストに収まるサイズに切ることで、「全部を知らなくても次の一手が打てる」

### PRD → tasks.md の粒度設計

タスクは「読むべきファイルが 1〜3 個で済む粒度」に切る。目安：

- 1 タスク = 1 ファイルの作成 or 修正
- テストは実装と同じタスクに含める
- 設計判断を伴うタスクは単独で切る（例：「AST ノードの基本構造を決定する」）

### Backlog — 「今じゃないが忘れたくない」の受け皿

Ralph Loop 実行中に Claude Code が「これもやった方がいい」と気づくことがある。しかし「1 イテレーション = 1 タスク」の原則を破って手を出すと中途半端な変更が残る。Backlog は以下を解決する：

- Claude Code は気づきを `tasks.md` の `## Backlog` に分類タグ付きで追記する
- **Backlog のアイテムを勝手にタスク化・実装することは禁止**
- Phase 境界で人間がトリアージし、次 Phase に昇格するか判断する
- 分類タグ: 🐛 bug / 💡 idea / 🔧 refactor / 📝 doc / ⚠️ debt

### µCC の教訓（テンプレートに反映済み）

- **位置情報は Phase 1 から入れる** — 後付けすると大規模リファクタリングが必要になる
- **型表現は最初から専用の型を使う** — 仮に `String` で表現すると技術的負債になる
- **デシュガーパターンを活用** — 新しい構文は既存構文への変換で済まないか最初に検討する

## カスタマイズ

### 言語別の調整

**Go プロジェクト:**
- `PROMPT.md` のビルドコマンドを `go build ./...` / `go test ./...` に変更
- `CLAUDE.md` に「外部依存ゼロ」「テーブルドリブンテスト」等を記述

**MoonBit プロジェクト:**
- `PROMPT.md` のビルドコマンドを `moon check` / `moon test` に変更
- `CLAUDE.md` に moonbit-practice プラグイン参照を追加

**その他の言語:**
- `PROMPT.md` の `BUILD_CMD` / `TEST_CMD` を書き換えるだけで対応可能
