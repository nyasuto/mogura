## 5. Ralph Loop 運用設計

### 5.1 ファイル構成

```
project/
├── ralph.sh
├── PROMPT.md
├── tasks.md
├── CLAUDE.md
├── docs/
│   └── PRD.md        # このファイル
└── ...
```

### 5.2 イテレーション設計

- 1 イテレーション = 1 タスク（tasks.md の 1 チェックボックス）
- Phase 完了時に次 Phase のタスクを tasks.md に展開
- ブロックされたタスクには `⚠️ BLOCKED: <理由>` を追記

### 5.3 Backlog 運用

- Claude Code がタスク実行中に気づいた改善点・バグ・アイデアは tasks.md の `## Backlog` に分類タグ付きで追記する
- Backlog のアイテムを Claude Code が勝手にタスク化・実装することは禁止
- Phase 境界で人間が Backlog をトリアージし、次 Phase のタスクに昇格するか判断する
- 急ぎのバグ（🐛）のみ、現在の Phase に差し込み可

### 5.4 人間の介入ポイント

<!-- どこで人間がレビュー・判断すべきかを明記 -->
- Phase 境界での設計レビューと Backlog トリアージ
- `⚠️ BLOCKED` が発生したタスクの判断
- 設計メモ（tasks.md 末尾）の確認と CLAUDE.md への反映
- Backlog の 🐛 bug が蓄積している場合の差し込み判断
