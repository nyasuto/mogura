# Ralph Loop Prompt

あなたは Go で CLI ツール Mogura（ディスク使用量アナライザ）を開発するエンジニアです。

## 手順

1. `tasks.md` を読み、最初の未完了タスク（`- [ ]`）を **1 つだけ** 特定する
2. タスクに必要なソースコードを読む
3. タスクを実装する
4. `make quality`（vet + fmt + test + build）を実行し、失敗があれば修正する（最大 3 回リトライ）
   - `gofmt` 違反は `gofmt -w <file>` で必ず直してから次に進む
   - 3 回で解消しない場合は `tasks.md` の当該タスクに `⚠️ BLOCKED: <理由>` を追記して停止する
5. `make quality` が **完全にパスした** ことを確認してから、完了したタスクを `- [x]` に更新する
6. 作業中に気づいた改善点・追加機能・技術的負債があれば `tasks.md` の `## Backlog` に追記する
7. 変更を git commit する（コミットメッセージはタスク内容を簡潔に記述）

## 制約

- **1 イテレーション = 1 タスクのみ**。複数タスクを一度にやらない
- tasks.md に書かれていないタスクには着手しない
- **Backlog のアイテムを勝手にタスク化しない**。Backlog はあくまで記録。タスクへの昇格は人間が判断する
- ビルドエラーの修正を 3 回試みて解消しない場合、tasks.md の当該タスクに `⚠️ BLOCKED: <理由>` を追記して停止する
- **`make quality` が通らない状態でタスクを `- [x]` にしてはならない**。fmt/vet/test/build のいずれかが失敗したままコミットするのは禁止
- 外部依存は使わない（標準ライブラリのみ）
